# Спецификация: Реализация конечной точки проверки работоспособности

## Описание
Данный трек предполагает реализацию базовой конечной точки HTTP GET `/health`, которая будет возвращать статус работоспособности сервиса. Эта конечная точка является стандартным инструментом для мониторинга состояния микросервисов в облачных средах и системах оркестрации (например, Kubernetes). Она позволит внешним системам быстро определять, запущен ли сервис и готов ли он принимать запросы.

## Цели
*   Создать HTTP GET конечную точку `/health`.
*   Конечная точка должна возвращать HTTP-статус 200 OK при успешном запуске сервиса.
*   Ответ должен включать простую полезную нагрузку, указывающую на статус сервиса (например, `{ "status": "ok" }`).
*   Конечная точка должна быть доступна без аутентификации.

## Нефункциональные требования
*   **Производительность:** Ответ должен быть быстрым, без задержек.
*   **Надежность:** Конечная точка должна всегда возвращать корректный статус, даже при высокой нагрузке на основной сервис.
*   **Безопасность:** Доступ к конечной точке должен быть открытым, без требований к авторизации, так как она предназначена для внешнего мониторинга.

## Предполагаемые компоненты
*   **Контроллер:** Новый контроллер или метод в существующем контроллере для обработки запроса `/health`.
*   **Сервис:** Возможно, потребуется простой сервис для формирования ответа о состоянии.
*   **Маршрутизация:** Настройка маршрута для `/health` в NestJS.
